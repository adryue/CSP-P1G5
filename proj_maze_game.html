MAZE GAME

Sep - Nov 2023





CREATION PROCESS

We decided to build this game using Snap4Arduino, as it seemed like a good introduction to learning how to work with Arduino. Our game is essentially a maze runner, where the player navigates through a series of mazes using an Arduino controller.

Early on, we first realized that in order to be able to control the player and have the camera follow the player around the maze, we needed to actually move the background itself instead of the player. This created essentially the same effect, where the player is centered on the screen while moving.

We created some placeholder maps that consisted of just black lines on a transparent background. In our code, we made it so that if the wall sprite detected the player sprite color, it broadcasted a message that would reset the game. Even though the code for resetting is only one block long, it will be useful later as we add more things.

We made several levels worth of maze maps using a drawing program. On each map is a yellow win zone that the player must reach in order to complete the map and immediately progress to the next one.

This was achieved by creating a new sprite for the landing zone and implementing the exact same code for movement, as it counts as part of the background. In our player code, we added a simple loop that broadcasted a message when it detects the yellow color of the win zone. This message would be picked up by the wall, which would then switch maps and reset the position. Each map is a different costume in the map sprite.

We started implementing an Arduino joystick to use as a controller in the game. In the breadboard configuration, we wired the ground and 5v to the power rails. For the joystick, we wired GND (the one next to U/D) to the ground power rail and U/D+ and L/R+ to the 5v power rail. We connected U/D to A1 (analog 1) and L/R to A2 (analog 2).

In Snap4Arduino, we read the joystick input as X and Y values from A2 and A1. The values range from 0 to 1023 (moving the joystick all the way to one side and then all the way to the other side), and the neutral position is around 500. So, we can shift the input down by 500 and treat it as the speed of the player (around -500 to 500).
