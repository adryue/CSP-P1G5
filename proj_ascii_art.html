CREATION PROCESS

This program was made using PIL (pillow), which stands for Python Imaging Library. To use it, you need to run the command “pip install pillow” in your terminal.

Instructions for use: Place the images in the folder “input_images” and run the command “python main.py input_images\[image.jpg]”, replacing [image.jpg] with your file name. The converted text is stored in the folder “output_text”.

PIL allows us to open an image from a file.

We can also manipulate the image by doing things like increasing the brightness (not necessary), boosting the contrast (for a more dramatic effect), and converting to grayscale (makes conversion to ASCII much easier, as now we only have to work with one light value instead of three: red, green, and blue).

We need to scale down the image to fit a specific number of characters horizontally (a constant value that we defined, usually 150 or 200 characters). The number of characters used vertically depends on the same constant: Take the ratio of height/width and multiply it by the number of characters. Since text characters are not perfectly square (they are slightly elongated), the image also needs to be shrinked vertically to compensate. We aren’t doing a 1:1 ratio for mapping pixels to characters. We’ll only scale it down enough so that each character represents a 4x4 region of pixels.

To actually convert the image to text, we iterate through every pixel region, take the average light value, and find the character that most closely matches that light value (we created our own character map).

Then, we can save the result into a text file.

To create our character map, we essentially need to take an image of every character that we use, add up all the light values in each of the characters, and sort them based on those light values. We’ll scale the light values so that the lowest one is 0 and the highest one is 255.
